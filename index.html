The Basics of HTML

This documentation was created with non-coders in mind who are interested in the basics necessary to get a website up and running. It assumes you have basic knowledge using a website.

Introduction

HTML stands for Hyper Text Markup Language, and it provides the foundation and basic structure of a webpage. It's the basic building block of the web. It allows us to break text into headers, paragraphs (for long-form writing), and other basic structures associated with, say, a word document. And, like in a word processor, you can make text bold, italic, or designate text as a quote pulled from another source.

HTML lets us manually mark up text in a document so the browser knows what to do with that text and how to display it. You begin by wrapping various text with tags, which are keywords and sometimes even single letters surrounded by angle brackets. HTML documents are connected to each other using links.

Anatomy of HTML

An HTML element consist of three parts: the opening tag, the content, and the closing tag

  • The opening tag - The name of the element (or sometimes just a single letter) surrounded by opening and closing angle brackets. In the example above, what we have is a paragraph tag.
  • The content - This is the content that will ultimately end up being displayed on the website, and it's what we're instructing our browswer to format via HTML tags
  • The closing tag - the closing tag ends our element, indicating where we want formatting to stop. Like the opening tag, it's written using two angle brackets, but it also includes a backslash.

Basic Set-Up

There's a few set of HTML elements that we must include in every website, and they operate as instructions for the browser, telling it that we're writing an HTML document and thus it needs to be rendered as such in the browser. Many developers save the following code as a snippet so they don't have to write it over and over again, and the popular editor plugin Emmet saves all of it to a keyword that will automatically populate your HTML document with the code upon using it.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="utf-8"> <title>My sweet website</title> </head> <body> <p>Hello, world!</p> </body> </html>

  1. !DOCTYPE - it used to be that an HTML needed to follow a specific set of rules to be considered good HTML, and the doctype acted as a link to those rules. You don't need to follow those rules or link to them, but this is the bare minimum we need to include to make our website work
  2. HTML - this is your root element. It wraps all of the content on your page
  3. HEAD - This wraps up information associated with your website, but isn't necessarily content that is viewable on your page. It's where you can put the title and keywords or a description for your website that will show up in search engines
  4. META - This specifies the character set that will be used on your document. In this case, we're using UTF-8, which includes most of the characters for the vast majority of human written languages.
  5. TITLE - The title of your website! It's what will appear in the tab of the browser or in a bookmark menu if someone adds the page to their bookmarks.
  6. BODY - This is the element that will wrap all of your viewable content. It's your working space where you'll add text, images, interactive media like videos, audio, etc.

An important HTML element that should be mentioned before we dive into the basic building blocks for a website is the <a> tag, which is used to transform text into a clickable element that leads you to another HTML document. These are called links and while it doesn't seem significant on its own, links are the glue of the Internet. They're what creates the "web" in World Wide Web!

<a href="www.google.com">Google</a> <a href="surprise.html">Click Me!</a> <a href="mailto:test@fakeemail.com">Send Me an Email</a>

The href that you see above is what is referred to as an attribute, and they append additional information to an HTML tag, and sometimes can alter behavior. In this case, the href attribute is what you use to assign a URL to a piece of text enclosed by the a tag.

*NOTE: If you're wondering, HREF is short for Hypertext REFerence.

Establishing Structure

Despite the wide variety of design seen in websites, most sites rely on the same standard components to provide structure and organize information. There are elements for the three components seen on just about any website you come across: the header, the main section, and a footer, and all three have their own tags!

<header> Welcome to my Website! </header> <main> I ♥ coffee, black metal, and coding </main> <footer> Copyright Jenny Mikac 2020 </footer>

The header is typically seen across the top of the website and is usually reserved for a logo and navigation, and sometimes navigation. It can also be used to designate the heading of an article or a new section of the document.

In the header, you can often find another handy HTML element, the nav element. It's used to designate navigation in some form or another (usually in the header or as a sidebar)

<nav> <a href="about.html">About</a> <a href="blog.html">Blog</a> </nav>

The main element, as indicated by the name, contains the main section of a website. This is where most of the content is. On a blog, this may contain all of the individual blog posts, and on a website like YouTube it will contain the video you're watching, the description and meta information, and the comment section.

Within the main section, you'll often find the article and section elements being used. You can use them to further break down your main content area. For example, on YouTube, one section could be the video itself, and another section would be the comments. It groups content together by theme. An article, on the other hand, is a piece of information that can be reused and presented on its own (a blog post, for example)

<section> Lorem ipsum dolor sit amet, consectetur adipiscing elit. </section> <article> Lorem ipsum dolor sit amet, consectetur adipiscing elit. </article>

Visually, neither of these tags make a difference when marking up an HTML document, but semantically, they're important for developers and browsers, as they each have an ideal use case.

The footer is located at the bottom of the page and, like on a document in the real world, may contain copyright information and fine print. Some of the information seen in the header may be repeated here, as well, and often companies will include links to their social media accounts.

A popular, catch-all HTML element that is often used when structuring and organizing content is the division or div tag. Before HTML5 and the addition of elements like header or article, div was the popular choice to divide up an HTML document.

Formatting Text

Just like when you're working on a document in Microsoft Word or Apple's Pages, you can format the appearance of your text to make it large, italicized, or bold. Here's a list of the the most commonly used HTML tags for formatting text.

  1. <b></b> or <strong></strong> is used to make text bold. The strong tag is preferred because it's more semantically correct (that is, the tag adds addictional meaning to the text)
  2. <i></i> or <em></em> will italicize your text. Like the strong tag above, the em tag is preferred.
  3. <u></u> will underline your text
  4. <pre></pre> defines preformatted text: this is text that uses a fixed-width font, and will also preserve spaces and line breaks. A good example of this is the code you see in the black boxes throughout this document.

You can also define headings of varying levels using the header tags. There are 6 of them, designating a hierarchy of information

<h1>This is a level 1 heading; it's the largest</h1> <h2>This is a level 2 heading</h2> <h3>This is a level 3 heading</h3> <h4>This is a level 4 heading</h4> <h5>This is a level 5 heading</h5> <h6>This is a level 6 heading; it's the smallest</h6>

When rendered in the browser, the above will look like this:

This is a level 1 heading; it's the largest

This is a level 2 heading

This is a level 3 heading

This is a level 4 heading

This is a level 5 heading
This is a level 6 heading; it's the smallest

HTML also makes it really easy to create pre-styled lists. The ones used the most are the unordered list (ul) and the ordered list (ol). You define your list by using one of the above tags, and then each individual item in your list must be then wrapped in the list (li) element.

<ul> <li>Milk</li> <li>Eggs</li> <li>Diapers</li> </ul>

The unordered list will look like this when rendered in the browser

  • Milk
  • Eggs
  • Diapers

And an ordered list...

<ol> <li>Sushi</li> <li>Pizza</li> <li>Bread</li> </ol>

...will look like this

  1. Sushi
  2. Pizza
  3. Bread
